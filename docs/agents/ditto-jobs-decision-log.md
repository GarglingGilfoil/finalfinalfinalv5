# Ditto Jobs Decision Log

Last updated: 2026-04-22  
Purpose: Keep Orchestrator decisions traceable across phases and routes.

## Usage Rules
- Log any decision that affects scope, architecture, contract shape, routing, acceptance criteria, route-level UX, route-level layout/composition, or route-level visual direction.
- Keep entries short and concrete.
- One decision per ID.
- If a decision is replaced, add a new entry and mark prior entry as `Superseded`.
- Route-level decisions should be logged when they are likely to drift during implementation or review.

## Decision ID Format
- `DJD-###` (for example `DJD-001`)
